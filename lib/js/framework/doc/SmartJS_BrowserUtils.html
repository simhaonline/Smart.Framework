Ajax_XHR_Request_From_URL static
(
  • y_url
  • y_method
  • y_data_type
  • y_data_arr_or_serialized
  • y_AuthUser
  • y_AuthPass
  • y_Headers
)
Object

Create a general purpose Ajax XHR Request (GET/POST) with Optional support for Authentication and Extra Headers It is NOT intended to be used with HTML forms that may contain multipart/form-data and file attachments or not. For creating an Ajax XHR Request to be used with HTML Forms use: SmartJS_BrowserUtils.Ajax_XHR_GetByForm(); // basic, will detect the form type if must use multipart/form-data + attachments (if any) or even much better use this: SmartJS_BrowserUtils.Submit_Form_By_Ajax(); // advanced, will handle the XHR form request and the answer

Parameters:

  • y_url String

    The URL to send the Request to

  • y_method Enum

    The Request Method: GET / POST

  • y_data_type Enum

    The type of Data served back by the Request: json | html | text

  • y_data_arr_or_serialized Mixed

    The Data to be sent: a serialized string via serialize() such as: '&var1=value1&var2=value2' or an associative array as: { var1: "value1", var2: "value2" }

  • y_AuthUser String

    Optional The Authentication UserName (if custom Authentication need to be used)

  • y_AuthPass String

    Optional The Authentication Password (if custom Authentication need to be used)

  • y_Headers String

    Optional Extra Headers to be sent with this Request

Returns:

Object:

The Ajax XHR Request Object ; The following methods must be bind to this object and redefined: .done: function(msg) {} .fail: function(msg) {} .always: function(msg) {}

    Optional the width of the Dialog (will be used just if UIDialog / SimpleDialog are detected) ; if not set the default width will be used

  • y_height Integer

    Optional the height of the Dialog (will be used just if UIDialog / SimpleDialog are detected) ; if not set the default height will be used

Confirm a Form Submit by a confirm / Dialog, with OK and Cancel buttons. The form will be submitted just if OK button is clicked. This function is using confirm_Dialog() and will detect and prefer in this order if UIDialog / SimpleDialog or just confirm() are available in Browser.

Parameters:

  • y_confirm String

    The question / message to display for confirmation (HTML)

  • y_form JS-Code

    The HTML Form Object to bind to

  • y_target String

    The window target to send this form to

  • windowWidth Integer

    Optional the width of the new Modal/PopUp child window if new window target is used ; if not set the default width will be used

  • windowHeight Integer

    Optional the height of the new Modal/PopUp child window if new window target is used ; if not set the default height will be used

  • forcePopUp Enum

    Optional if a new target window is required, 0 = default, use modal/iFrame if set by SmartJS_BrowserUtils_Use_iFModalBox_Active, 1 = force PopUp ; -1 force modal/iFrame

  • forceDims Enum

    Optional if set to 1 will try force uwing the width/height set for the new modal/iFrame or PopUp
